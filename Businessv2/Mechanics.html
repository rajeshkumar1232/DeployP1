Automobile Repair Guide Project Outline

1. User Roles

  • Admin: Manages users, workshops, repairs, and tools catalog.
  • Mechanics/Users: Follow instructions on how to repair vehicles.
  • Workshop Owners: Manage workshop details, assign mechanics, and monitor repairs.

2. Core Features

a. Problem Identification and Diagnosis

  • Frontend (React): Users can search or select common problems (e.g., "engine overheating", "brake issues").
  • Backend (Spring Boot): Retrieves:
    • Location in the car where the issue is likely occurring.
    • Steps on how to investigate and diagnose the issue.
    • Tools required for the job.
    • Possible causes of the problem.
  • Database: Stores repair instructions, problems, and tools.

b. Repair Instructions

  • Frontend (React): Displays step-by-step repair instructions with illustrations or diagrams.
  • Backend (Spring Boot): Serves instructions based on the selected problem.
  • Tools Integration: Shows required tools, how to use them, and their relevance to the task.

c. Interactive Car Diagram

  • Frontend (React): Displays an interactive diagram of the car that highlights where to check based on the problem.
  • Backend (Spring Boot): Supplies the data for the highlighted areas.

d. Tool Catalog

  • Frontend (React): Mechanics and users can view a list of tools for each task.
  • Backend (Spring Boot): Admin can manage the tool catalog and link tools to specific repair jobs.

e. Workshops and Repairs Management

  • Frontend (React):
    • Mechanics/Users: Log in, see assigned tasks, and track repair completion status.
    • Workshop Owners: Manage mechanics, assign tasks, and view progress.
  • Backend (Spring Boot): Handles user authentication, task assignments, and repair tracking.

3. Architecture Overview

a. Frontend (React)

  • Build UI components for problems, car diagrams, repair instructions, and tool management.
  • Ensure mobile-friendliness with responsive design.
  • Use Redux or Context API for state management.

b. Backend (Spring Boot)

  • Expose RESTful APIs for:
    • Problem retrieval (problem list, details, location).
    • Repair steps retrieval.
    • Tool catalog management.
    • Workshop and repair management.
    • User roles and authentication.
  • Use a relational database (MySQL/PostgreSQL) for storing repair instructions, tools, and users.
  • Implement role-based access control using Spring Security.

4. Database Schema

  • Users Table: Stores user details and roles (Admin, Mechanic, Workshop Owner).
  • Problems Table: Contains car problems, symptoms, and possible solutions.
  • Repair Steps Table: Steps to fix a problem, tools, and time estimates.
  • Tools Table: Tool name, description, usage, and linked repair steps.
  • Workshops Table: Workshop details and associated mechanics.

5. Technologies to Use

  • Frontend: React, Redux/Context API, Material-UI/Bootstrap, D3.js or Three.js for car diagrams.
  • Backend: Spring Boot, JPA/Hibernate, Spring Security.
  • Database: MySQL or PostgreSQL.
  • 3D Modeling: Use libraries like Three.js for car diagrams.

6. Key Use Cases

  • Search/Select Problem: Users search for a problem or select from a pre-populated list.
  • View Instructions: Users follow guided repair instructions with images and tool usage tips.
  • Admin Tool Management: Admin can add new tools and associate them with repair tasks.
  • Workshop Management: Workshop owners assign tasks to mechanics and track progress.

7. Advanced Features (Optional)

  • Implement AI-powered diagnosis to suggest the most likely issues based on user inputs or symptoms.
  • Consider Augmented Reality (AR) to overlay repair instructions on real-world images of the car.
  • Implement a notification system to inform mechanics about new tasks or repair updates.

8. Workflows

  • Admin Flow: Admin logs in → Manages problem database, repair instructions, and tools.
  • Mechanic/User Flow: Logs in → Selects a problem → Follows guided repair instructions → Updates task status.
  • Workshop Owner Flow: Logs in → Manages workshop details → Assigns tasks → Monitors task completion.

9. Security and Authentication

  • Use JWT tokens for user authentication and session management in both frontend and backend.
  • Ensure role-based access control for Admin, Mechanic, and Workshop Owner roles.

10. CI/CD and Deployment

  • Frontend: Deploy the React app on platforms like Netlify or Vercel.
  • Backend: Deploy the Spring Boot application on AWS, GCP, or Heroku.
  • CI/CD Pipelines: Use GitHub Actions, Jenkins, or CircleCI for automated testing and deployment.
